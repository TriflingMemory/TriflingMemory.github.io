Distributed likes

Written by Trifling Memory

Random thoughts: Using crypto-concurrency to store data.

Distributed likes

Most social websites resolve around finding the good little bits hidden around the internet and showing them to the people who might like them.

This is normally done using a like/up-vote/stumble/dig or whatever button. A central authority collects all the likes/votes/stumbles/dugs or whatever and processes them.

We could replace that system by assigning a unique crypto-currency address to every piece of distributed content. Sending crypto-currency to that address equals a upvote. It is now up to any 3th party to aggregate the “likes” and provide suggestions to similar content.

Enabling likes for your digital content requires you to do two things. Generate a new unique crypto-currency address and paste this address on your content. Like this (look at the lower left corner of the comic):

You should use a different address for every piece of likeable content. There are exceptions for this, if you distribute the same content in different formats or if you distribute a new minor version of the same content. You are allowed to use the same address in those cases. Deviate work from a different artist MUST always use a different address.

In addition pasting this address you need to sign your content. This facilitates address -> content lookups. I am not about the best way to do this. It might look like this:

Everything between the BEGIN and END block is signed. This signed message does not need to be visible to any user. It only needs to be indexed by the search engines.

The good:

  • No single central authority can influence the votes.
  • Any digital content can be “like-able”. Sound/text/links.
  • Duplicated content on different sites (which still has the same associated address) still receives upvotes in the same bucket.

I consider that last point its greatest benefit.

The bad:

  • Everybody can influence the votes.
  • Crypto-currency was not designed as global upvote system. I used dogecoin address above, but there might exist more suited currencies.
  • Doing a reverse search (crypto-currency address to content) is not simple. You need to employ a search engine and hope the content provider put the address in plain text.
  • No analysis and aggregation software for something like this exist. Creating this software requires tackling a few hard challenges.
  • You need crypto-currency software and crypto-currency to like something.

A lot of research needs to be done if this is feasible.
